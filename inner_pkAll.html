UX for a client-facing platform

The project

On a daily basis, PrimeKey PKI clients deal with hundreds of thousands of issued certificates, trust handshakes, revocation lists, access management, passport verification and digital signatures â€“ all through web app interfaces. While the backend Java code is state of the art, the product frontend was built on deprecated Java technologies and was incredibly difficult to use, even for experienced system administrators. Additionally, PrimeKey products did not follow a uniform layout and visual strategy to promote the brand.

My role:

Research

Ideation

Prototyping

User Testing


I was tasked with researching an information architecture overhaul for all PrimeKey products, simplifying an extremely complex interfaces, cost effective interactive prototypes and wireframes, as well as recommendations to standardize the installation procedure across all PrimeKey products.

In this project my technical experience was of immediate help - it helped with establishing a good relationship with Java developers to whom User Centered Design was a novelty. The result of the project was an extensive usability report, concrete solution wireframes, a visual style guide, and a roadmap for implementing the findings.

Challenges

The platform is designed to cater to a wide variety of clients, ranging from small business owners to corporates and institutions. This requirement mandated that every solution can support different expectations and workflows, which was an important focus for the team. We also had to ensure that the different solutions are not siloed, but instead work as an ecosystem aware of different moving parts.
